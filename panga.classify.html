class panga.classify.read_rules.SubRule(string)[source]¶

Bases: object

A Subrule will receive a string with structure:

(part1, operand, part2)

where part1 and part2 will be compared using the operand. Either part could be a number, a pure string (such as ‘pore’, ‘user1’) or a keyword refering to an attribute of the metrics (e.g. median_current, pore_before).

The result will be an object that has done the parsing of the parts and the operand, and can be used to apply the rule it represents.

evaluate(metrics)[source]¶

Evaluate the comparison of the rule. If the elements are keywords meant for the metric, use them as such.

Parameters:metrics – mapping object containing the metrics of the reads.
Returns:boolean indicating if the reads given pass the current subrule
requires¶

Required attributes are all those participating in the comparisons that are not numbers or literally strings to compare with. They should be part of the metrics, and will be called when evaluating.

Returns:a list of the required keywords.
